# 社会治理--网格化 SocialGovernance
以电子地图和网格化管理为基础，整合各类网格资源、管理资源、服务资源、软件系统资源和数据资源，把人、地、事、物、组织、流程、服务等各类基础要素纳入整体的社会管理和服务体系中，做到治理无盲区、无缝隙、全覆盖。“智治”大脑作为整个城市治理的核心大脑，所有系统和数据能够在中心集中展现，并根据现场情况进行调度、协调、指挥；通过视频会议进行会商决策、调度指挥，减轻各级部门之间的沟通代价，提高工作效率。通过历城项目的建设，形成社会治理的新方式、新途径，构建起基于大数据、大治理、大一体的城市发展新环境。
网格化信息服务管理平台，即社会治安综合治理管理系统。根据属地管理、地理布局、现状管理等原则，将管辖地域划成若干网络状的单元，并对每一网络实施动态、全方位管理。按照对等方式整合公共服务资源，组织服务团队，对网格内的居民进行多元化、精细化、个性化服务。 从而实现城市管理具备监控、反馈、督办功能，可避免城市的“死角”和“盲点“，是一种事前预警性管理。从而提高管理水平，提高政府和居民的良性互动。
# 网格化信息服务管理平台（社会治安综合治理管理系统）

#### 目标
1. 建立“基层为主、四级联动”的矛盾调处和群众权益协调模式
2. 建立“化被动为主动”的群众诉求获取和社会矛盾源头治理模式
3. ”精细化、责任化”的社会管理统筹协调模式
4. 建立“监管分离、考管结合”的社会管处胃保障模式
5. 建立人、地、物、事、组织、房屋、网格等城市基础数据综合管理和可持续更新模式
6. 建立“听群众说、向群众讲、带群众干、请群众评、让群众享”的网格治理模式。

   
#### 系统优势

1.  、建设社会治理数据中心，实现基础数据汇集
2. 建设二、三绯地图，实现关键细节“一张图全学控”的目标
3. 建设应急指挥平台，实现实时调度指挥
4. 建设“街道吹哨、部门报到”工作机制，规范工作流程
5. 建设实时数据的采集和传输，保障综治业务的实时高效


该平台管理公务人员、公众账号等各类用户基本数据、身份数据以及权限数据，通过基于身份数据的4A（Access Control、Account、Authenticaton、Authorization）标准服务，建立统一、开放、安全的认证平台，能够为平台的所有用户提供统一的身份确认与权限交付。
通过统一身份认证，改善用户感知，提供差异化服务，通过帐号、业务的整合，为用户提供全应用系统的通行服务。
事项处置中台的集中受理需要有一个跨部门、跨区域的统一身份认证体系作为基础。山东省已经建立统一身份认证平台，实现了统一授权、统一认证，只需登录一次，全网事项通办。区域内各受理系统应按照省统一身份认证接入规范接入全省统一用户体系。
依托基层工作人员和服务团队工作内容，梳理社会治理综合信息指标，内容主要包括辖区信息、常住人口、流动人口、重点人口、重点场所、社会组织等相关信息。协调相关部门数据，建立社会治理综合信息数据库。根据社会治理业务需求，提供相对应的数据管理功能。
通过基层录入，整合公安、司法、民政等部门业务系统，形成完善的社会治理信息数据库，确保人口、场所、组织处于有效的监管之下。
利用信息化手段对综治组织建设及其他综合业务进行采集、统计汇总，动态掌握各级综治组织和队伍建设情况，模块包括：综治机构、综治队伍、群防群治组织、群防群治队伍、综治中心、综治视联网信息中心、网格化建设、楼栋长、综治领导责任制、重特大案（事）件。
综治机构提供维护辖区内的综治信息，包括机构名称、机构类型和机构层级等。
1.1.2综治队伍
综治队伍维护综治队伍下的工作人员信息，包括姓名、性别、所属机构、民族、政治面貌、公民身份证号、出生日期、级别、职务、专业特长、学历、手机号码和固定号码等。
1.1.3群防群治组织
维护的群防群治组织信息包括组织名称、组织类型、组织层级、人员数量、指导业务部门、主要职能、所属网格等信息。
1.1.4群防群治队伍
群防群治队伍的维护信息有姓名、性别、所属机构、民族、政治面貌、证件类型、证件号码、出生日期、职务、专业特长、学历、手机号码和固定号码等信息。
1.1.5综治中心
综治中心提供维护综治中心名称、联系方式、综治中心层级、负责人姓名、负责人联系方式、组成单位和所在地等信息。

1.1.6综治视联网信息中心
综治视联网信息中心提供维护综治中心名称、联系方式、综治中心层级、负责人姓名、是否24小时有人值班、专职工作人员数量、办公用房面积、年度运行经费、联网公共安全视频监控摄像机数量、所在地、所在地详细地址和标绘地址等。


2.制定标准处置流程 
事项处置中台的集中受理和分拨处置需要遵循统一的事项处理流程和标准。首先,要优化当前历城区各部门的工作流程,消除冗余环节,明确必要的审批节点和时限要求。在此基础上制定跨部门、全流程的标准处理规范,作为中台业务集成的依据。不同部门对接中台时,需要调整自身流程适应统一标准,以保证处置的规范性。
根据属地管理、地理布局、现状管理等原则，将管辖地域划分成若干网格状的单元，并对每一网格实施动态、全方位管理。对网格基础信息进行管理，实现社会服务管理平台所需的基础数据功能。利用GIS系统对各级网格进行标绘划分，达到全区行政区域无缝隙、全覆盖。
网格划分：根据属地管理、地理布局、现状管理等原则，将管辖地域划分成若干网格状的单元，以便于工作的分配和相关巡查人员的巡查。信息也做到网格化的分区，便于信息的分别统计和汇总。
网格展现：结合GIS地图进行展示。

对于已有的行政区划，在地图上将其管理区域展现出来，若行政区划还未与天地图进行关联，则可在天地图上进行标绘操作。
标记类型:通过上传不同的图标对各类场所在地图上进行标记展示。

2.1事件管理
事件管理主要功能是通过群众来信来访、电话热线、手机终端、政民互动、居民诉求等多种渠道，记录群众的各方面反应和诉求，并按照职能对事件进行分流，由相关部门和人员进行处理解决。该功能全程记录每件事件的受理、处理、办结及反馈评价的全过程。
按照“琐事不出网格、小事不出社区/村、大事不出街道/乡镇”的建设目标，各级社会治理工作人员协调处理由基层反映上来的卫生计生、社会保障、社会治安、城市管理、社会事务、环境卫生、安全生产、道路交通、矛盾排查、食品食药、消防安全等各类事件，进行及时上报反馈、派遣、调处到最终的结案，避免问题的扩大化，并且能与GIS地图进行联动。
社会治理人员在处理事件的时候，可以根据事件类型、难易程度选择自行处理或上报给上级或其他部门进行处理。提供快速查询功能，以便工作人员按照涉及人员、场所等信息搜索事件。
2.2事件分类管理
进行事件类型管理，可将事件结合实际业务需求进行分类，如民生、环境卫生、道路交通等，同时可将民生、环境卫生等类别进行细化出小类，更加贴合实际。对事件类别进行灵活管理。
2.3事件来源管理
对事件来源进行分类，如群众来信来访、电话热线、居民诉求等类别，用户可根据需求进行新增、修改。
2.4事件流程处理
事件流程举例：网格员上报（或自行处理）—指挥中心分派—专业部门处理—网格员现场复核—网格中心审查（若审核不通过可驳回重新办理）—结案。（当事件发展到一定程度时可转为矛盾，转向矛盾纠纷流程）。
详尽的历史流程跟踪记录、标准的流程和执行规范、职责明确，轻松追溯、清晰掌控事件全生命周期。实现对事件处理的过程进行监督，对处理不及时的事件进行督办。

2.5事件上报
将呼叫中心接收、政民互动、来访来信件等方式反馈的事件信息填写录入，或网格员将巡查现场事件信息及时录入上报。按照事件类型，紧急程度结合业务流程进行分流上报处理。结合GIS地图功能可标绘出事件发生位置，支持图片、视频、音频等多种格式附件上传，确保事件信息的准确性，尽量还原事件现场的真实度。可查看上报的事件信息列表，点击事件可查看详情，及事件处理情况，及时了解上报的事件处理进度。

2.6审批配置
除以上举例事件流程外，用户可根据业务流程灵活配置事件流程节点，自由选择流程去向进行分办、协办等办理方式，支持多种业务流程配置。

3.实现全程智能监控
中台需要整合各部门的事项处理全链路,对处理进度等进行智能化监控。如果出现节点延误、数据缺失等异常情况,中台可以快速进行预警提醒。全程监控可以大幅提升事项处置的效率,防止在部门之间或节点之间“卡顿”的情况发生。
3.1全部工单
查看系统授权范围内的所有事件信息，包含授权区域内自己办理和其他人办理的事件信息。
统计汇总：按照时间、行政区划、事件类型、事件来源等多维度进行统计，通过柱状图、饼状图、折线图、报表等方式展现。为领导决策提供直观依据和数据依据。
趋势分析：将管辖区域内所有事件进行汇总，通过报表形式按照行政区划中事件总数、已办事件、未办事件、超期事件排行。直观了解个地区事件上报及处理情况，并结合现实情况做出工作重点调整。
3.2辖区巡查
网格员打开手机端巡查，系统通过GPS定位功能，将社会治理工作人员每次巡查路线行成轨迹，记录巡查路线及距离。方便领导掌握基层工作人员每天的工作情况。
当网格开启巡查功能时，网格员可通过视频直播功能向领导回传巡查现场情况，可及时撑握网格巡查情况。同时方便领导对基层网格人员进行考勤，实现现场点名。
3.3视频监控
视频监控主要功能是通过整合有效社会资源，达到资源信息共享最大化。视频监控逐步实现治安动态视频接入、交通视频接入、安全生产视频接入和社会视频接入，形成属地管理、区域管理的综合监控系统。结合GIS地图功能，展示视频监控分布情况，方便快速查找、调取监控录像。
设置统一的监控中心，设计监控中心统一的人员监控管理位置，并充分考虑防辐射、监控环境等建设内容，形成统一的、唯一的监控中心。
监控整合平台，具有集成度高、兼容性强、稳定性强、便于管理维护和使用等优点。
3.4场所管理
实现机关管理、企业管理、商超管理、医院管理基本信息维护。可进行查看、新增、删除、修改、查询等操作。
3.5日常工作
日常工作主要功能是记录辖区范围内的网格人员考勤打卡情况、辖区巡情情况、入户走访情况、任务下发及执行情况，提供工作日记记录功能，方便工作人员做好日常工作情况记录。
工作日志服务，维护、查看社会治理工作人员的日常工作日志（非系统操作日志），比如工作人员对重点场所的巡查情况。
3.6考勤打卡
为社会治理工作人员提供考勤功能，通过手机端进行上签到、下班签退。
3.7信息发布
政策信息公开、公示公告、文件规章、政策法规、政务实事等内容在信息发布平台进行发布，用户通过系统平台和手机客户端查看。
3.8入户走访
当网格员入户走访时，记录每户走访信息。更全面的掌握家庭成员的基本情况，提升人民服务质量。

3.9任务下发
领导可根据各级部门、网格工作情况，通过移动应用向下级网格下发工作任务。下发对象包括机构、部门、人员，可以选择单个对象下发任务，也可以同时向多个对象下发任务。 如：指定部门进行定点巡查社区、重点场所，处理重点突发事件等。
创建任务可添加图片、文档等附件，设置任务完成期限、及任务紧急程度。
领导可随时跟踪任务完成进度，对已经完成的任务进行审核。若对任务完成情况不满意可驳回重新办理。
3.10任务执行
接收到领导安排的任务，根据任务情况进行任务办理。

3.11领导工作台
通过饼状图、柱状图、分布图等多种图形灵活展示管辖网格内各类人、事件、场所所占比例、分布等情况。通过饼状图分析占比情况、通过柱状图分析同比和环比增长情况、通过曲线图分析趋势。可以根据区域或者类型并按照时间区间进行分析。方便领导快速了解所管辖网格区域内的人、事件、场所的情况，把握工作重心、抓住重点信息、做出具体决策。
3.12监督考核
监督考核依据网格员工作情况，同时结合系统其它有关数据，将综治考核和业务考核的定量标准在网上实现，以手工打分和系统自动打分相结合模式，对本系统要求完成的工作任务进行网上实施监督评估。
考核项设置，允许各级社会治理领导为下级社会治理部门设置工作考核标准，包括考核项（事件处理次数、隐患调处数量、满意度、考勤打卡、巡查路线等）、每项分数等规则。
主要考核内容：对网格员日常出勤情况、入户走访情况、巡查情况、事件办理数量、矛盾调处数量等各项工作内容进行考核。
4.智能分拨事项
中台在受理事项后,需要根据部门业务权限、工作负载等因素智能分配事项。此举一方面提高分配的精准性,另一方面也实现资源的动态优化配置。如果仅靠人工分配,很难达到这样的精细化管理。智能分拨可以大幅提升事项流转的效率。
4.1督导督办
督导督办：领导根据各个部门事件处理情况进行督导督办、做出重要批示，被督办部门登录系统后进行消息提醒，对督办事件信息优先排序，确保事件及时有效处理。
督办查询：查询督办的事件信息。
4.2综合查询
综合查询为用户提供灵活的查询功用，结合基础网格数据，综合展现居民信息，例如通过筛选条件查询某人信息，查询到的结果是整个平台中与之关联的数据（房产信息、是否重点关注人群、是否涉及某些事件等）。
具有数据钻探功能，查询结果可对数据信息进行深层次的挖掘，环环相扣，从而实现闭环查询。
结合地图提供直观图形化功能，使信息的展示更加直观。可通过输入查询条件，按照人、场所、事件分类进行查询并在地图上显示。可按照地图标绘功能，在地图上通过标绘出一片区域查询该区域内的人、场所信息、发生的事件信息。并将人、场所、事件信息进行关联。如：点击人物信息，可查看人的基本信息并查看与该人关联的场所信息（公司、企业信息等）、事件信息。

人物查询：输入查询条件进行快速查询，列表显示相关人员信息，并显示与人员关联的场所（公司、企业等）数量、事件数量。点击可查看人员详细信息及有关的场所详细信息、事件详细信息及事件处理信息。

场所查询：输入查询条件进行快速查询，列表显示相关场所信息，并显示与场所关联的人员数量、事件数量。点击可查看场所详细信息及有关的人员详细信息、事件详细信息及事件处理信息。
事件查询：输入查询条件进行快速查询，列表显示相关事件信息，并显示与场所关联的人员数量、场所数量。点击可查看事件详细信息及有关的人员详细信息、场所详细信息。
4.3已办事件
查看已经处理过的事件信息，可根据标题、内容等条件进行快速检索定位。

4.4待办事件
接收到流程流转的事件，根据事件类型及紧急程度进行排序，优先处理紧急事件。将事件处理工作变的更加条理有序。
4.5指挥中心（视频直播）
以信息化平台建设为基础，以构建统一指挥、联网联动工作机制为核心，以指挥调度、指令下达、资源调配、综合管理为职能，形成统一指挥、集中管控、多级联动的综合指挥体系。

手机APP调用：选择事件发生地，结合GIS地图功能沿事件发生地就近位置标绘一个区域范围，查看离事件发生地就近的巡查员，进行紧急派遣。巡查员利用手机App的现场直播功能将现场情况实时传送到指挥大厅，领导通过传回的视频画面做出紧急方案，通过App中的交流平台对现场做出指示。
